#Chat API Usage Instructions

## 1. Start a Chat Session

### Endpoint

```
POST /api/chat
```

### Description

Sends a user message to the AI. The API uses:

* vector database (RAG retrieval)
* previous chat history (memory)
* LLM (Gemini) to generate response

### Request Body (JSON)

```json
{
  "sessionId": "user123",
  "message": "Tell me about Sigiriya"
}
```

### Response

```json
{
  "success": true,
  "sessionId": "user123",
  "response": "Sigiriya is an ancient rock fortress in Sri Lanka...",
  "history": [
    { "role": "user", "text": "Tell me about Sigiriya" },
    { "role": "assistant", "text": "Sigiriya is..." }
  ]
}
```

---

## 2. Get Chat History

### Endpoint

```
GET /api/chat/history/:sessionId
```

### Description

Returns full conversation history for a given session.

### Example Request

```
GET /api/chat/history/user123
```

### Response

```json
{
  "success": true,
  "sessionId": "user123",
  "history": [
    { "role": "user", "text": "Tell me about Sigiriya" },
    { "role": "assistant", "text": "Sigiriya is..." }
  ]
}
```

---

## 3. How Chat Flow Works (Important)

1. Frontend sends `message + sessionId`
2. Backend:

   * retrieves relevant knowledge (RAG)
   * uses chat history for context
   * generates AI response
3. Response is returned + stored in session memory

---

## Important Notes

* `sessionId` is required to maintain conversation memory
* Each user should have a unique sessionId (e.g. user ID or UUID)
* Chat history is stored in-memory (resets if server restarts)
* Use `/chat/history` to restore UI state on refresh

---

## Frontend Tip

* Call `/api/chat` on every user message
* Append response to chat UI immediately
* Optionally load history on page load using `/chat/history`

---

2. Set Image Context (After Image Upload)

Endpoint
POST /api/chat/set-image-context
Description

Used after /api/image-search. This stores the identified image subject for contextual chat.

Request Body
{
  "sessionId": "user123",
  "imageContext": "Sigiriya Rock Fortress in Sri Lanka"
}

“Who built it?”
“Why is it famous?”

  "sessionId": "user123",
  "message": "Where is it located?"
}


The system automatically uses the stored image context.

Endpoint
GET /api/chat/history/:sessionId
Description

Returns full conversation history and image context.

Example
GET /api/chat/history/user123

Response
{
  "success": true,
  "history": [
    { "role": "user", "text": "What is Sigiriya?" },
    { "role": "assistant", "text": "Sigiriya is..." }
  ],
  "imageContext": "Sigiriya Rock Fortress in Sri Lanka"

5. System Flow Overview

Text-only flow:
User → /chat → RAG retrieval → AI response → history stored

Image-based flow:
Image upload → /image-search → identifiedSubject
             → /set-image-context
             → /chat (Ask more)
             → AI uses image context + RAG


Important Notes

sessionId is required for all chat-related requests
Image context is optional but enhances responses
Chat history is stored in-memory (resets on server restart)
Always call /set-image-context before asking follow-up image questions


Frontend Tip
Store sessionId in localStorage
After image upload → call /set-image-context
Enable “Ask More” button → redirects to chat page with same sessionId
Load history using /chat/history/:sessionId on page load
